# Gemini CLI モデル選択 (`/model` コマンド)

Gemini CLI モデルを選択します。`/model` コマンドは、Gemini CLI で使用されるモデルを設定できるダイアログを開き、結果をより細かく制御できるようにします。

**注:** `/model` コマンド（および `--model` フラグ）は、サブエージェントが使用するモデルをオーバーライドしません。したがって、`/model` フラグを使用しても、モデル使用量レポートには他のモデルが表示される場合があります。

## `/model` コマンドの使用方法

Gemini CLI で次のコマンドを使用します。

```
/model
```

このコマンドを実行すると、モデルオプションを含むダイアログが開きます。

| オプション             | 説明                                                   | モデル                                                                                     |
| ------------------ | ------------------------------------------------------------- | ------------------------------------------------------------------------------------------ |
| 自動（推奨）       | システムにタスクに最適なモデルを選択させます。           | gemini-3-pro-preview (有効な場合)、gemini-2.5-pro、gemini-2.5-flash、gemini-2.5-flash-lite |
| Pro                | 深い推論と創造性を必要とする複雑なタスク向けです。 | gemini-3-pro-preview (有効な場合)、gemini-2.5-pro                                          |
| Flash              | 速度と推論のバランスを必要とするタスク向けです。         | gemini-2.5-flash                                                                           |
| Flash-Lite         | 迅速に完了する必要があるシンプルなタスク向けです。                | gemini-2.5-flash-lite                                                                      |

### Gemini 3 Pro とプレビュー機能

注: Gemini 3 は現在、すべてのアカウントタイプで利用できるわけではありません。Gemini 3 アクセスの詳細については、[Gemini CLI での Gemini 3 Pro](../get-started/gemini-3.md) を参照してください。

Gemini 3 Pro を有効にするには（利用可能な場合）、[**設定**コマンドを使用してプレビュー機能](..//cli/settings.md)を有効にします。有効にすると、Gemini CLI は**自動**または**Pro**を選択したときに Gemini 3 Pro を使用しようとします。**自動**と**Pro**の両方で、Gemini 2.5 Pro にフォールバックする前に Gemini 3 Pro を使用しようとします。

また、`--model` フラグを使用して、起動時に特定の Gemini モデルを指定することもできます。詳細については、[設定ドキュメント](./configuration.md)を参照してください。

これらの設定の変更は、Gemini CLI とのすべてのその後のやり取りに適用されます。

## モデル選択のベストプラクティス

- **デフォルトは自動（推奨）です。** ほとんどのユーザーにとって、「自動（推奨）」モデルは速度とパフォーマンスのバランスを提供し、タスクの複雑さに基づいて正しいモデルを自動的に選択します。例: Web アプリケーションの開発には、複雑なタスク（アーキテクチャの構築とプロジェクトのスキャフォールディング）と単純なタスク（CSS の生成）が混在する可能性があります。

- **希望する結果が得られない場合は Pro に切り替えます。** モデルがもう少し「賢い」必要があると思う場合は、Pro を使用してください。Pro は、最高レベルの推論と創造性を提供します。例: 複雑な、または多段階のデバッグタスク。

- **より迅速な結果が必要な場合は Flash または Flash-Lite に切り替えます。** シンプルな応答を迅速に必要とする場合は、Flash または Flash-Lite が最適なオプションです。例: JSON オブジェクトを YAML 文字列に変換する。
