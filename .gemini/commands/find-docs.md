# Find Docs

**説明**  
ユーザーの問いに応じた関連ドキュメントをリポジトリ内から探し、GitHub で直接参照できる URL を一覧で提示するコマンドです。

## ミッション：関連ドキュメントを見つける

1. Git などでリモート URL を取得し、HTTPS/SSH 形式どちらも扱って base URL（例: `https://github.com/user/repo`）を構築。デフォルトブランチは `main` と想定する。
2. 質問に関連しそうな `.md`/`.mdx` などのドキュメントを対象に、まずは絞り込んで検索。ヒットしなければ `docs/` 以下のすべてを読み、さらに必要ならキーワードの類義語も含めて幅広く調査する。
3. 有力候補は実際に内容を読んで妥当性を確認。
4. 見つかった各ファイルについて `{BASE_URL}/blob/{BRANCH}/{path}` 形式の GitHub URL を構成し、Markdown のリストとして提示。URL にコマンドは使わない。
5. それでも見つからなければ、URL は提示せずユーザーへ補足質問を投げ、意図を明確化してもらう。

## 出力
- 各候補を「URL — 内容の要約」の形で Markdown リストにまとめる。
