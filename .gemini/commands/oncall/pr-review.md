# PR Review

**説明**  
指定された PR を慎重にレビューして安全性や品質を確認し、マージ可否を判断するためのワークフローを示します。

## ミッション：PR を徹底レビューする

1. 指定された PR（`{{args}}`）をチェックアウトし、一時ブランチで調査。
2. `npm run preflight` を実行してビルド・リンティング・ユニットテストを走らせ、出力結果に失敗や警告がないか確認。
3. PR 差分を詳細に解析し、以下の観点で評価する。
   - 正当性（意図通り動くか）  
   - 保守性（構造が明快・再利用しやすいか）  
   - 可読性（コメントやフォーマットが統一されているか）  
   - 効率性（パフォーマンスやリソース消費への配慮）  
   - セキュリティ（脆弱性を生み出していないか）  
   - エッジケース・例外処理の充実度  
   - テストカバレッジの適切さ（不足がないか）  
4. 過去のレビューコメント・要求が解決済みか確認し、必要なら現状に触れて明示。
5. マージ可能なら、品質を称賛する丁寧な承認コメントを作成（前提チェックには触れない）。  
6. マージ不可なら、具体的で建設的な問題と改善案を列挙し、貢献者が対応できるようにする。
7. すべてのレビュー作業後、希望する手動テストが終了したらその旨を伝える。  
8. 最後に `main` に切り替えて不要ブランチを削除、最新版を `pull` して次のレビュー準備を整える。
