# Gemini CLI

Gemini CLI 内では、`packages/cli` は、ユーザーが Gemini AI モデルとその関連ツールを使用してプロンプトを送受信するためのフロントエンドです。Gemini CLI の概要については、[メインドキュメントページ](../index.md)を参照してください。

## 基本機能

- [**コマンド**](./commands.md): すべての組み込みスラッシュコマンドのリファレンス
- [**カスタムコマンド**](./custom-commands.md): 頻繁に使用するプロンプト用の独自のコマンドとショートカットを作成します。
- [**ヘッドレスモード**](./headless.md): スクリプト作成と自動化のために Gemini CLI をプログラムで利用します。
- [**モデル選択**](./model.md): CLI で使用される Gemini AI モデルを構成します。
- [**設定**](./settings.md): CLI の動作と外観のさまざまな側面を構成します。
- [**テーマ**](./themes.md): さまざまなテーマで CLI の外観をカスタマイズします。
- [**キーボードショートカット**](./keyboard-shortcuts.md): ワークフローを改善するためのすべてのキーボードショートカットのリファレンス。
- [**チュートリアル**](./tutorials.md): 一般的なタスクのステップバイステップガイド。

## 高度な機能

- [**チェックポイント**](./checkpointing.md): セッションとファイルの変更履歴を自動的に保存および復元します。
- [**エンタープライズ構成**](./enterprise.md): エンタープライズ環境で Gemini CLI をデプロイおよび管理します。
- [**サンドボックス**](./sandbox.md): セキュアなコンテナ化された環境でツール実行を分離します。
- [**テレメトリ**](./telemetry.md): 使用状況とパフォーマンスを監視するために可観測性を構成します。
- [**トークンキャッシュ**](./token-caching.md): トークンをキャッシュすることで API コストを最適化します。
- [**信頼できるフォルダ**](./trusted-folders.md): CLI の全機能を使用できるプロジェクトを制御するセキュリティ機能。
- [**ファイルの無視 (.geminiignore)**](./gemini-ignore.md): 特定のファイルとディレクトリがツールによってアクセスされるのを防ぎます。
- [**コンテキストファイル (GEMINI.md)**](./gemini-md.md): モデルに永続的な階層的コンテキストを提供します。

## 非対話モード

Gemini CLI は非対話モードで実行でき、スクリプト作成と自動化に役立ちます。このモードでは、入力を CLI にパイプし、CLI はコマンドを実行してから終了します。

次の例は、ターミナルから Gemini CLI にコマンドをパイプする方法を示しています。

```bash
echo "What is fine tuning?" | gemini
```

`--prompt` または `-p` フラグを使用することもできます。

```bash
gemini -p "What is fine tuning?"
```

ヘッドレス使用法、スクリプト作成、自動化、および高度な例に関する包括的なドキュメントについては、[**ヘッドレスモード**](./headless.md)ガイドを参照してください。
