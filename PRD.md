# Product Requirements Document: Gemini CLI 日本語ドキュメント化

## 1. はじめに
本ドキュメントは、Gemini CLI プロジェクトのドキュメント（READMEおよびdocs配下）を日本語化し、日本の開発者コミュニティへの普及と利用促進を図るための仕様を定義する。

## 2. スコープ
*   **入力（翻訳対象）**:
    *   ルートディレクトリの `README.md`
    *   `docs/` ディレクトリ配下の全ての `.md` ファイル（サブディレクトリを含む）
*   **出力（成果物）**:
    *   `README_J.md` (ルートディレクトリに新規作成)
    *   `docs_j/` ディレクトリおよびその配下の翻訳済みファイル群

## 3. 詳細仕様

### 3.1 ディレクトリ構造とファイル命名
*   `docs/` ディレクトリと同階層に `docs_j/` ディレクトリを新規作成する。
*   `docs_j/` 内部には、`docs/` 内部のディレクトリ構造を完全に再現する。
*   翻訳後のファイル名は、元の英語ファイル名と同一とする（例: `docs/index.md` → `docs_j/index.md`）。`_j` などの接尾辞は付与しない。

### 3.2 翻訳品質とガイドライン
*   **翻訳スタイル**: 原文の文法構造や意味を正確に伝える「直訳」を基本とする。過度な意訳は避ける。
*   **専門用語**: プログラミング言語、フレームワーク、ツール名、コマンド名などの技術用語（例: `npm`, `Vitest`, `React`, `Gemini CLI`, `Plan Mode`）は、カタカナ表記にせず英語のまま維持する。
*   **コードブロック**: コードブロック内のコード、コマンド、出力例は翻訳しない。ただし、コードブロック内のコメントについては、文脈理解に必須であれば日本語化してもよい。

### 3.3 リンクの整合性確保
*   **README_J.md**:
    *   ファイル内に記述されている `docs/` ディレクトリ以下のファイルへのリンクを、`docs_j/` ディレクトリ以下の対応するファイルへのリンクに置換する。
    *   例: `[Architecture](docs/architecture.md)` → `[Architecture](docs_j/architecture.md)`
*   **docs_j/ 配下のファイル**:
    *   ファイル間の相対リンクについては、ディレクトリ構造が維持されているため、原則として修正を行わない。

### 3.4 既存資産の保護
*   フォーク元および既存の `README.md`、`docs/` ディレクトリ内のファイルは読み取り専用として扱い、いかなる変更も加えない。

## 4. 開発プロセス
1.  `docs_j` ディレクトリの作成と構造複製。
2.  `README.md` の翻訳と `README_J.md` の生成。
3.  `docs/` 内のファイルのリストアップと順次翻訳、`docs_j/` への配置。
4.  リンクの動作確認。
